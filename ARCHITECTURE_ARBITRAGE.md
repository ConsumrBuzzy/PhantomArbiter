# Arbitrage Engine Architecture (V1.0)

## Overview

The Arbitrage Engine is a **high-frequency spread detection and execution system** for Solana DEXs.
It monitors price discrepancies across multiple venues and executes profitable trades automatically.

## ğŸ¯ Core Design Principles

1. **Visibility First**: Real-time Console and Telegram updates are the primary interface
2. **Deterministic Profits**: Only execute when math guarantees profit after all fees
3. **Atomic Execution**: Multi-leg trades succeed or fail together (no partial fills)
4. **Low Latency**: Sub-second opportunity detection and execution

## ğŸ”„ Main Loop

```python
async def arbitrage_loop():
    """Main arbitrage engine loop."""
    
    while True:
        # 1. Fetch prices from all DEXs
        prices = await price_aggregator.fetch_all()
        
        # 2. Detect opportunities
        opportunities = spread_detector.scan(prices)
        
        # 3. Display live dashboard (Console + Telegram)
        dashboard.update(prices, opportunities)
        
        # 4. Filter profitable opportunities
        for opp in opportunities:
            if risk_manager.validate(opp):
                # 5. Execute trade
                result = await executor.execute(opp)
                
                # 6. Log and notify
                tracker.record_trade(result)
                await telegram.notify(result)
        
        # 7. Brief cooldown
        await asyncio.sleep(1)  # 1 second tick
```
